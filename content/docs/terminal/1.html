Hilfe

Kurzhilfe für die Verwendung eines Befehls kann man in der Regel auf unterschiedliche Art abfragen:

  • Windows: git /?
  • Linux & Mac: git --help oder git -h

Syntax

In der Hilfe und an dieser Stelle wird die Benutzung oft in einer vereinfachten BNF-Syntax angegeben, die sich an folgendem Muster orientiert: {{< highlight bash>}}befehl [optional] {{< / highlight>}} Optionale Flags und Parameter werden in eckigen Klammern, Platzhalter eigene Parameter in spitzen Klammern angegeben.

Windows (PowerShell)

Command Prompt / Befehlszeile:

{{< highlight powershell>}}C:\Users\user>{{< /highlight>}} Windows zeigt den Laufwerksbuchstaben und das Arbeitsverzeichnis an.

Linux & Mac (Terminal / zsh bzw. bash)

Command Prompt / Befehlszeile:

{{< highlight zsh>}}>user@mymac ~ %{{< /highlight>}} {{< highlight bash>}}>user@mylinux:~${{< /highlight>}} Linux und Mac zeigen den Benutzernamen und den Hostnamen an, dann kommt das Arbeitsverzeichnis. Besonderheit: das Home-Verzeichnis des eingeloggten Benutzers wird als ~ abgekürzt.

Kommandos

{{< highlight powershell>}}> befehl /flags parameter{{< /highlight>}} In Windows werden Flags mit einem Schrägstrich / gekennzeichnet. PowerShell akzeptiert an vielen Stellen inzwischen auch die Unix-Schreibweise mit Bindestrich -.

Kommandos

{{< highlight bash>}}>$ befehl -flags parameter{{< /highlight>}} In Linux und Mac werden Flags mit einem Bindestrich - gekennzeichnet.
